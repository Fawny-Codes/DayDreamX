Deployment Guide

Get DayDream X running on your server

⚠️ Important Note

DayDream X cannot be hosted on static web hosting services such as Netlify, GitHub Pages, or Cloudflare Pages. You need a server that can run Node.js.

Local Installation

To run DayDream X locally on your machine:

  1. Clone the repository:
    git clone https://github.com/NxroProxy/DayDreamX.git
  2. Navigate to the project directory:
    cd DayDreamX
  3. Install dependencies (choose your preferred package manager):
    npm i

    or

    bun install

    or

    pnpm install
  4. Start the development server:
    npm start

    or

    bun start

    or

    pnpm start

The browser will be available at http://localhost:3000 (port may vary).

Other Platforms

You can also deploy to:

  • Railway: Connect your GitHub repository
  • Render: Simple Git integration
  • DigitalOcean: Deploy on Droplets or App Platform
  • AWS: Using EC2, App Runner, or Elastic Beanstalk
  • Self-hosted: Deploy to any VPS or dedicated server

Updating

To update to the latest version:

git pull --force --allow-unrelated-histories

Package Managers

DayDream X supports multiple package managers:

  • npm: Node Package Manager (included with Node.js)
  • pnpm: Fast, space-efficient package manager
  • bun: Fast all-in-one JavaScript runtime (recommended)

DayDream X includes lock files for all three package managers, so choose whichever you prefer.

System Requirements

  • Node.js: v14 or higher
  • RAM: Minimum 512MB (1GB+ recommended)
  • Storage: Minimum 500MB free space

Troubleshooting

If you encounter issues:

  • Make sure Node.js is properly installed: node --version
  • Clear node_modules and reinstall: rm -rf node_modules && npm install
  • Check that required ports are available
  • Review logs for error messages
  • Open an issue on GitHub
